# 需求转PRD与原型提示词流水线

## 触发词

`执行需求梳理`

## 流水线概述

本流水线用于将原始业务需求、会议纪要或零散想法，结构化为标准 PRD，并自动提取用于 Axhub Make 生成高保真原型的 Prompt。

## AI 协同顺序与执行逻辑

### 步骤 1：明确需求名称

- **任务**：当接收到 `执行需求梳理` 触发词时，必须先明确“需求/功能名称”。如果用户没有提供，请先询问用户。
- **输出**：确认的需求/功能名称。

### 步骤 2：需求提炼与结构化

- **技能**：@prd
- **任务**：读取用户提供的原始上下文，输出包含业务背景、角色权限矩阵和数据字典的标准化 Markdown 文档。
- **输出**：结构化需求文档（草稿）。

### 步骤 3：逻辑闭环设计

- **技能**：@requirements-to-detailed-design
- **任务**：针对步骤 2 的文档，强制使用 Mermaid 语法绘制状态机和业务流转图，确保异常分支和边界条件无遗漏。
- **输出**：带有时序图和流程图的完整 PRD。

### 步骤 4：生成 Axhub 原型 Prompt

- **技能**：@requirements-closure-designer (或自定义 @axhub-prompt-generator)
- **任务**：基于定稿的 PRD，提炼出专供 Axhub Make 读取的页面布局与 UI 组件描述 Prompt。
- **输出**：Axhub 专属 Prompt 文本。

### 步骤 5：文件命名与存储

- **任务**：根据当前日期和确认的需求/功能名称，生成标准化的文件名，并存储在固定的业务目录中：
  - PRD 路径：`projects/[项目名称]/docs/YYYYMMDD-[功能名称].md`
  - Prompt 路径：`projects/[项目名称]/prompts/YYYYMMDD-[功能名称]-Prompt.md`
- **输出**：生成的文件路径。

### 步骤 6：IDE 焦点控制

- **任务**：文件生成完毕后，调用编辑器的能力，自动在主视窗打开刚刚生成的 `YYYYMMDD-[功能名称].md` 文件，以便用户进行确认和手动编辑。
- **输出**：编辑器打开的文件。

### 步骤 7：标准接力提示语与自动引用

- **任务**：在完成上述动作后，输出标准提示词。
- **输出**：标准接力提示语。

### 步骤 8：环境自动唤醒

- **任务**：在完成所有文件生成后，主动调用内置终端 (Terminal)，依次执行：
  - `npm run switch [当前项目名称]`
  - `npm run dev:axhub-make`
- **输出**：终端执行结果。

## 预期成果

- 严谨的 Markdown 格式 PRD，存储在 `projects/[项目名称]/docs/YYYYMMDD-[功能名称].md`。
- 可视化的 Mermaid 业务流程图。
- 可直接复制到 Axhub Make 生成原型的精确提示词，存储在 `projects/[项目名称]/prompts/YYYYMMDD-[功能名称]-Prompt.md`。
- 自动打开的 PRD 文件，方便用户编辑确认。
- 标准的接力提示语，引导用户进行下一步操作。
- Axhub 引擎自动启动，准备就绪。

## 标准接力提示语

```
✅ 需求文档和 Prompt 已生成。已为您打开 `YYYYMMDD-[功能名称].md`，并成功生成了 `prompts/YYYYMMDD-[功能名称]-Prompt.md`。如果您修改了需求文档，请回复 `执行Prompt同步`；如果确认无误，请回复 `执行产研流水线`。
🚀 Axhub 引擎已为您启动，请点击上方 Localhost 链接打开浏览器，将 Prompt 粘贴至界面中进行高保真原型生成。
```

## 执行条件

- 用户提供了明确的需求描述
- 目标项目目录已存在
- 网络连接正常

## 注意事项

- 确保需求/功能名称清晰明确，避免使用模糊或过于宽泛的名称
- 文件名必须严格遵循 `YYYYMMDD-功能名称.md` 和 `YYYYMMDD-功能名称-Prompt.md` 格式
- 生成的 PRD 文档应包含完整的业务背景、功能需求、用户角色、数据字典和业务流程
- 生成的 Prompt 应详细描述页面布局、UI 组件和交互逻辑
