# CTIS Cloud 代码审查标准

## 快速开始

### 代码审查清单

#### 功能性检查
- [ ] 代码实现了需求文档中的所有功能
- [ ] 代码逻辑正确,没有明显的bug
- [ ] 边界条件处理正确
- [ ] 异常情况处理完善
- [ ] 业务逻辑符合需求

#### 代码质量检查
- [ ] 代码命名符合规范
- [ ] 代码结构清晰,易于理解
- [ ] 代码复杂度合理
- [ ] 没有重复代码
- [ ] 代码可读性好

#### 性能检查
- [ ] 数据库查询使用了索引
- [ ] 避免了N+1查询
- [ ] 使用了缓存优化
- [ ] 没有性能瓶颈
- [ ] 资源使用合理

#### 安全性检查
- [ ] 输入参数进行了校验
- [ ] SQL注入防护
- [ ] XSS攻击防护
- [ ] 敏感信息已加密
- [ ] 权限控制正确

#### 测试检查
- [ ] 单元测试覆盖率达标
- [ ] 测试用例完整
- [ ] 测试代码质量高
- [ ] 集成测试通过
- [ ] 测试用例有意义

#### 文档检查
- [ ] API文档完整
- [ ] 代码注释充分
- [ ] README文档更新
- [ ] 变更日志记录

### 审查流程

1. **自我审查**: 提交前先自我审查
2. **同行审查**: 至少1-2名同事审查
3. **架构审查**: 涉及架构变更时需要架构师审查
4. **安全审查**: 涉及安全相关代码时需要安全专家审查
5. **测试审查**: 确保测试覆盖充分

### 审查反馈规范

- **建设性**: 提供建设性的改进建议
- **具体**: 指出具体的问题位置
- **友好**: 保持友好的沟通态度
- **及时**: 及时响应审查请求
- **解释**: 对拒绝的变更给出解释

## 常见问题检查点

### 代码规范问题
- 命名不符合规范
- 缺少必要的注释
- 代码格式不统一
- 魔法数字未定义为常量

### 性能问题
- 循环内执行数据库查询
- 未使用索引的查询
- 大对象频繁创建销毁
- 未使用缓存

### 安全问题
- SQL注入风险
- XSS攻击风险
- 敏感信息明文存储
- 权限校验缺失
