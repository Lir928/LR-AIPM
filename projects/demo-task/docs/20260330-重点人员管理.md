# 重点人员管理需求文档

## 1. 业务背景

在基层管理中，重点人员管理是一项重要的工作，需要对企业内部的关键人员进行集中管理和监控。这些重点人员可能包括高级管理人员、核心技术人员、敏感岗位人员等，他们对企业的运营和发展具有重要影响。

## 2. 功能需求

### 2.1 人员信息管理

- 支持重点人员信息的录入、编辑、删除和查询
- 支持人员信息的批量导入和导出
- 支持人员信息的分类管理

### 2.2 人员状态监控

- 实时监控重点人员的状态变化
- 支持状态异常的预警和通知
- 支持状态历史记录的查询和分析

### 2.3 权限管理

- 基于角色的权限控制
- 支持不同角色对重点人员信息的访问权限设置
- 支持操作日志的记录和查询

### 2.4 统计分析

- 支持重点人员数据的统计和分析
- 支持生成各类统计报表
- 支持数据可视化展示

## 3. 用户角色

| 角色     | 权限                     |
| ------ | ---------------------- |
| 系统管理员  | 全部权限，包括人员管理、权限设置、系统配置等 |
| 部门主管   | 管理本部门的重点人员信息，查看相关统计报表  |
| 人力资源专员 | 录入和管理重点人员信息，查看基本统计数据   |
| 普通用户   | 查看授权范围内的重点人员信息         |

## 4. 数据字典

### 4.1 人员信息表

| 字段名        | 数据类型   | 校验规则         | 默认值  | 是否必填 |
| ---------- | ------ | ------------ | ---- | ---- |
| id         | string | 唯一标识符        | 无    | 是    |
| name       | string | 长度 2-50      | 无    | 是    |
| gender     | string | 枚举值：男/女      | 无    | 是    |
| age        | number | 18-65        | 无    | 是    |
| department | string | 长度 2-100     | 无    | 是    |
| position   | string | 长度 2-100     | 无    | 是    |
| level      | string | 枚举值：A/B/C/D  | 无    | 是    |
| status     | string | 枚举值：正常/异常/离职 | 正常   | 是    |
| entryDate  | date   | 日期格式         | 无    | 是    |
| lastUpdate | date   | 日期时间格式       | 当前时间 | 是    |

### 4.2 状态变更表

| 字段名        | 数据类型   | 校验规则         | 默认值  | 是否必填 |
| ---------- | ------ | ------------ | ---- | ---- |
| id         | string | 唯一标识符        | 无    | 是    |
| personId   | string | 关联人员ID       | 无    | 是    |
| oldStatus  | string | 枚举值：正常/异常/离职 | 无    | 是    |
| newStatus  | string | 枚举值：正常/异常/离职 | 无    | 是    |
| changeTime | date   | 日期时间格式       | 当前时间 | 是    |
| operator   | string | 操作人          | 无    | 是    |
| reason     | string | 长度 0-500     | 无    | 否    |

## 5. 业务流程

### 5.1 人员信息管理流程

1. 录入重点人员信息
2. 审核人员信息
3. 发布人员信息
4. 定期更新人员信息

### 5.2 状态监控流程

1. 实时监控人员状态
2. 发现状态异常
3. 发送预警通知
4. 处理异常情况
5. 记录处理结果

### 5.3 权限管理流程

1. 创建用户角色
2. 分配权限
3. 应用权限设置
4. 定期审查权限
