# 测试用例模板

## 用例格式

| 字段 | 说明 |
|------|------|
| 用例ID | TC-{模块}-{序号}，如 TC-LOGIN-001 |
| 模块 | 功能模块名称 |
| 用例名称 | 简洁描述测试场景 |
| 前置条件 | 测试执行前需要满足的条件 |
| 操作步骤 | 编号的详细步骤 |
| 预期结果 | 期望的系统响应/行为 |
| 优先级 | P0(关键路径) / P1(核心功能) / P2(边界情况) / P3(异常处理) |

## 示例

### TC-LOGIN-001: 用户登录成功

| 字段 | 内容 |
|------|------|
| 模块 | 用户登录 |
| 用例名称 | 用户使用正确凭证登录成功 |
| 前置条件 | 1. 用户已注册<br>2. 系统正常运行 |
| 操作步骤 | 1. 打开登录页面<br>2. 输入用户名<br>3. 输入密码<br>4. 点击登录按钮 |
| 预期结果 | 1. 页面跳转至首页<br>2. 显示用户欢迎信息 |
| 优先级 | P0 |

### TC-LOGIN-002: 用户登录失败-密码错误

| 字段 | 内容 |
|------|------|
| 模块 | 用户登录 |
| 用例名称 | 用户使用错误密码登录失败 |
| 前置条件 | 1. 用户已注册<br>2. 系统正常运行 |
| 操作步骤 | 1. 打开登录页面<br>2. 输入用户名<br>3. 输入错误密码<br>4. 点击登录按钮 |
| 预期结果 | 1. 显示错误提示"密码错误"<br>2. 停留在登录页面 |
| 优先级 | P0 |
