# API接口联调调试Skill

## 简介
这是一个使用chrome-devtools-mcp工具进行API接口联调测试的技能包。通过自动化访问Swagger文档页面，帮助开发者快速进行接口测试和调试。

## 功能特性
- ✅ 自动打开Swagger文档页面
- ✅ 根据模块名称定位接口分组
- ✅ 交互式选择要测试的接口
- ✅ 自动填写和执行接口请求
- ✅ 实时查看响应结果
- ✅ 错误诊断和建议

## 快速开始

### 激活Skill
在对话中使用以下命令激活此skill：
```
请激活api-debug-skill技能
```

### 使用示例
```
用户: 我要联调用户管理模块的接口

AI: [自动执行以下步骤]
1. 打开Swagger文档页面
2. 定位用户管理模块
3. 展示接口列表
4. 引导选择接口
5. 执行测试并展示结果
```

## 支持的接口文档格式
- Swagger UI (OpenAPI 2.0/3.0)
- Knife4j (增强版Swagger)
- 其他基于Swagger的文档系统

## 系统要求
- 可访问的Swagger文档地址
- chrome-devtools-mcp工具已配置
- 网络连接正常

## 使用场景
1. **开发阶段**: 快速测试自己开发的接口
2. **联调阶段**: 与后端协作验证接口功能
3. **问题排查**: 定位接口调用失败的原因
4. **回归测试**: 验证接口修改后的正确性

## 注意事项
- 确保Swagger文档地址可访问
- 某些接口可能需要先登录获取认证token
- 填写参数时注意数据格式和类型
- 建议在测试环境使用，避免影响生产数据
