# Figma 转前端代码

本 Skill 提供完整的 Figma 设计稿转前端代码工作流程，适用于 Vue 3 项目。

## 适用场景

- 用户要求根据 Figma 设计稿实现前端页面
- 用户要求像素级还原设计视觉效果
- 用户要求将设计稿转换为代码

## 核心流程

### 第一步：生成 AI 编程指导文档

1. 使用 AI 工具解析项目框架
2. 生成标准化的 AI 编程指导文档（AGENTS.md）
3. 文档内容包括：
   - 编程语言与框架
   - UI 组件库
   - 核心依赖与版本约束
   - 项目目录结构说明
   - 样式规范
   - 构建与部署流程

### 第二步：导入 Figma 设计资源

1. 在 Trae 中启用 **Solo 模式**
2. 启用 **Figma 插件**
3. 在 Figma 设计稿中选中目标页面或组件区域
4. 点击 **"添加到对话"** 按钮
5. AI 将自动下载切图资源到 `.figma` 目录
6. 将切图资源整理到 `src/assets/images/XXXX` 目录

### 第三步：编写定制化提示词

根据项目实际技术栈和设计风格编写提示词，参考以下模板：

```
请严格依据项目 .figma 文件夹中提供的 Figma 设计稿原件（含示例代码），在 src\views\[目标目录] 目录中像素级还原前端页面的视觉效果与交互结构。具体要求如下：

1. 视觉还原要求
   切图资源：设计稿中需要的切图资源已经下载到 .figma\image 中
   精准还原：必须完全按照 Figma 设计稿中的布局、层级、间距、颜色值、字体、切边框、圆角图、阴影、、渐变等所有视觉细节进行实现
   禁止臆测或自创：不得在未参考设计稿的情况下自行添加、修改或推测任何样式或布局逻辑。如有设计稿未覆盖的边界情况，需明确标注并暂停开发，等待确认

2. 代码实现规范
   figma代码仅供你参考，不得复用：Figma 中附带的示例代码仅用于理解布局结构，不可直接复制使用
   遵循项目规范：所有代码必须严格遵守根目录下 AGENTS.md 文件中定义的编码规范
   组件化拆分：必须将页面按功能和视觉模块拆分成多个独立、职责单一的组件文件，避免将所有逻辑和 UI 堆砌在单一文件中
```

### 第四步：创建 AI 工作计划

使用 Plan 模式让 AI 生成分步骤的开发任务计划，开发者审核后执行。

## 校验与调整技巧

### 技巧1：视觉一致性调整

1. 人工比对 AI 输出与设计稿
2. 使用 Trae 浏览器模式精准修正
3. 发出具体可量化的修改指令：
   - "将主按钮的圆角从 4px 改为 8px"
   - "卡片内上下 padding 调整为 16px，左右为 24px"
   - "标题文字颜色改为 #1F2937，字体大小 18px"

### 技巧2：创建项目 SKILLS

将重复性经验沉淀为可复用的 SKILLS：

1. **样式覆盖 SKILL**：当某个 UI 库样式修改多次才成功时
2. **组件使用 SKILL**：业务组件标准化使用说明

### 技巧3：图表高保真还原

使用 ECharts 时：

- 完全复刻配色、坐标轴样式、图例位置、网格线、tooltip 样式
- 支持 hover 高亮、数据缩放、图例切换等交互
- 使用响应式配置

### 技巧4：复杂 Bug 调试

使用 `/plan` 指令让 AI 先输出调试方案：

- 分析可能原因
- 列出排查步骤
- 遇到不清楚的地方务必停止并等待确认

## 组件化拆分原则

1. 按功能模块拆分组件
2. 每个组件职责单一
3. 保持代码结构清晰
4. 命名语义化
5. 逻辑解耦
6. 高度整洁性、可读性和可维护性

## 项目规范遵循

所有代码必须严格遵守：

- AGENTS.md 中的编码规范
- 组件命名规范（PascalCase）
- 文件命名规范（kebab-case）
- 样式规范（Less/SCSS）
- API 接口规范
