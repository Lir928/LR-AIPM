# Frontend Design Generator

## Overview

本技能通过分析当前项目的代码框架、组件库使用习惯、样式规范等，自动生成符合项目标准的前端详细设计文档。采用模块化设计，将复杂功能拆分为独立模块，使用task工具并行生成各模块设计文档，确保文档的可执行性和代码生成的准确性。

## 核心工作流程

### 阶段1：项目分析与需求理解

1. **读取项目配置和代码结构**
   - 分析package.json、vite.config.js等配置文件
   - 识别技术栈（Vue 3/React、UI组件库、状态管理等）
   - 提取项目代码风格和命名规范
   - 识别现有组件和API调用模式

2. **理解输入需求**
   - 文档：读取需求文档、设计稿、原型图
   - 图片：分析UI设计图，提取布局和交互信息
   - 文字：解析功能描述，识别核心需求

3. **功能模块划分**
   - 根据需求识别独立功能模块
   - 分析模块间的依赖关系
   - 确定生成顺序（基础模块优先，依赖模块后置）
   - 生成功能模块清单

### 阶段2：并行生成设计文档

使用task工具为每个功能模块调用subagent并行生成设计文档：

**每个subagent的任务：**
1. 读取references中的设计模板和规范
2. 根据模块需求生成详细设计内容
3. 确保与项目代码风格一致
4. 输出独立的Markdown设计文档

**生成顺序控制：**
- 第一批：基础组件、通用工具、布局结构
- 第二批：业务功能模块（无依赖）
- 第三批：依赖其他模块的功能

### 阶段3：文档整合与验证

1. **检查依赖关系**
   - 验证模块间的引用关系正确性
   - 确保组件命名不冲突
   - 检查API接口定义一致性

2. **生成主索引文档**
   - 创建设计文档总览
   - 列出所有模块及其依赖关系
   - 提供代码生成指导

## 设计文档内容规范

每个功能模块的设计文档应包含：

### 1. 模块概述
- 模块名称和功能描述
- 业务目标和用户价值
- 与其他模块的关系

### 2. 页面布局设计
- 整体布局结构（使用Ant Design Vue组件）
- 响应式设计说明
- 关键区域划分和尺寸规范

### 3. 组件设计
- 组件层次结构
- 组件职责划分
- 组件接口定义（props、events、slots）
- 复用组件的引用

### 4. 数据流设计
- 状态管理方案（Pinia store设计）
- 数据来源和API接口
- 组件间数据传递方式
- 表单验证规则

### 5. 交互设计
- 用户操作流程
- 交互反馈机制（loading、success、error）
- 动画和过渡效果
- 异常处理流程

### 6. 样式规范
- 主题色和配色方案
- 字体和排版规范
- 间距和布局规则
- Less/CSS类命名规范

### 7. 代码实现指导
- 文件结构建议
- 关键代码示例
- 注意事项和最佳实践
- 测试要点

## 使用方法

### 基本用法

```
用户：根据这个需求文档生成前端详细设计
[提供需求文档路径或内容]

Claude：
1. 分析项目代码框架
2. 读取需求文档
3. 划分功能模块
4. 并行生成各模块设计文档
5. 输出设计文档总览
```

### 高级用法

**指定技术栈：**
```
用户：使用Vue 3 + Ant Design Vue生成用户管理模块设计
```

**指定生成顺序：**
```
用户：先生成基础组件设计，再生成业务模块
```

**引用现有组件：**
```
用户：复用项目中已有的Table组件生成列表页设计
```

## 注意事项

1. **文档大小控制**
   - 单个模块文档不超过5000字
   - 复杂模块拆分为多个子文档
   - 使用引用而非重复内容

2. **依赖关系管理**
   - 明确标注模块依赖
   - 优先生成基础模块
   - 避免循环依赖

3. **代码风格一致性**
   - 严格遵循项目现有规范
   - 使用项目已有的组件和工具
   - 保持命名风格统一

4. **可执行性**
   - 设计文档应可直接指导代码生成
   - 包含具体的实现细节
   - 提供代码示例
