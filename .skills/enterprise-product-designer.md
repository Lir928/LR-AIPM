# 企业级 B 端产品设计规范

## 核心原则

### 1. 列表驱动原则
- 所有页面设计必须以列表为核心，清晰展示数据结构
- 列表必须包含：序号、核心字段、操作按钮
- 支持排序、筛选、搜索功能
- 分页显示，默认每页 20 条数据

### 2. 多步骤弹窗规范
- 复杂操作必须使用多步骤弹窗
- 每个步骤必须有清晰的标题和描述
- 步骤指示器必须显示当前进度
- 支持前进、后退、取消操作
- 最终步骤必须有确认操作

### 3. ASCII 绘图规范
- 必须为每个核心页面绘制 ASCII 布局图
- 布局图必须包含：页面标题、导航栏、操作区、数据展示区、分页控件
- 使用 ASCII 字符绘制边框和分隔线
- 清晰标注每个区域的功能

### 4. 字段级描述表格
- 为每个表单和数据展示页面创建字段级描述表格
- 表格必须包含：字段名、数据类型、校验规则、默认值、是否必填
- 数据类型必须明确：字符串、数字、日期、布尔值、枚举等
- 校验规则必须详细：长度限制、格式要求、范围约束等

## PRD 输出要求

### 1. 页面结构
- 每个页面必须包含：页面名称、URL、功能描述、页面布局图
- 页面布局图必须使用 ASCII 绘制
- 明确页面的响应式设计考虑

### 2. 字段设计
- 每个字段必须有详细的描述表格
- 字段名必须使用驼峰命名法
- 数据类型必须符合数据库设计规范
- 校验规则必须可直接转换为前端验证逻辑

### 3. 交互流程
- 复杂操作必须使用流程图表示
- 多步骤操作必须详细描述每个步骤的输入输出
- 异常情况必须有明确的处理流程

### 4. 权限设计
- 明确每个页面和操作的权限要求
- 基于角色的权限控制设计
- 权限检查点的具体位置

## 格式规范

### 1. 标题层级
- 使用 Markdown 标题层级，最多 4 级
- 标题必须清晰反映内容结构

### 2. 表格格式
- 使用 Markdown 表格语法
- 表头必须加粗
- 表格内容必须对齐

### 3. ASCII 图表
- 使用 ASCII 字符绘制图表
- 确保图表在不同终端下显示正常
- 图表必须有标题和说明

### 4. 代码块
- 使用 Markdown 代码块语法
- 代码块必须有语言标识
- 代码格式必须规范

## 强制要求

1. 严格按照本规范输出 PRD
2. 必须包含 ASCII 布局图
3. 必须包含字段级描述表格
4. 所有表格必须完整填写
5. 所有流程必须清晰描述
6. 所有格式必须符合规范

## 质量标准

1. PRD 必须可直接用于 Axhub 原型生成
2. 字段描述必须可直接用于数据库设计
3. 校验规则必须可直接用于前端验证
4. 布局图必须准确反映页面结构
5. 流程描述必须完整覆盖所有场景
