# 子端并行开发编排

## 触发词
`执行多端同步`

## 流水线概述
本流水线用于协调多个子端的并行开发，确保各端之间的 API 接口和通用组件配置保持同步，提高开发效率和一致性。

## AI 协同顺序

### 步骤 1：并行代理调度
- **技能**：@dispatching-parallel-agents
- **任务**：调度多个 AI 代理并行处理不同子端的开发任务
- **输出**：并行工作的 AI 代理

### 步骤 2：子代理驱动开发
- **技能**：@subagent-driven-development
- **任务**：让 AI 分别进入 `apps/web-workbench` 和 `apps/citizen-mobile`，同步更新两端的 API 接口或通用组件配置
- **输出**：同步更新的多端代码

## 执行条件
- 多端开发已启动
- 各端代码结构已建立
- 网络连接正常

## 预期成果
- 各端 API 接口保持同步
- 通用组件配置一致
- 开发效率显著提高

## 注意事项
- 确保各端代码结构相似，便于同步
- 同步过程中注意避免代码冲突
- 同步后进行测试，确保功能正常
