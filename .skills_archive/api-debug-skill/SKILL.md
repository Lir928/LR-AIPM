# API接口联调调试Skill

## 描述
使用chrome-devtools-mcp工具访问Swagger接口文档页面(http://172.30.0.28:31501/doc.html)，根据用户指定的模块进行接口联调测试。

## 使用场景
当需要测试后端API接口时，通过此skill可以：
1. 自动打开Swagger文档页面
2. 根据用户指定的模块定位相关接口
3. 执行接口调用并查看响应结果
4. 辅助进行接口联调

## 执行步骤

### 步骤1: 打开Swagger文档页面
使用`new_page`工具打开接口文档地址：
```
URL: http://172.30.0.28:31501/doc.html
```

### 步骤2: 等待页面加载完成
使用`wait_for`工具等待页面关键元素加载：
```
等待文本: ["Swagger UI", "API文档", "接口列表"]
```

### 步骤3: 获取页面快照
使用`take_snapshot`工具获取当前页面内容，分析页面结构

### 步骤4: 根据用户输入的模块定位接口
- 使用`take_snapshot`获取页面快照
- 分析快照中的模块分类和接口列表
- 根据用户指定的模块名称，找到对应的接口分组

### 步骤5: 展开目标模块
- 使用`click`工具点击目标模块的展开按钮
- 等待模块展开完成

### 步骤6: 选择要测试的接口
- 获取展开后的接口列表快照
- 向用户展示该模块下的所有接口
- 等待用户选择要测试的具体接口

### 步骤7: 执行接口调用
- 点击"Try it out"按钮
- 填写必要的请求参数（如果有）
- 点击"Execute"执行请求
- 使用`take_snapshot`或`list_network_requests`查看响应结果

### 步骤8: 分析响应结果
- 检查响应状态码
- 查看响应数据
- 判断接口是否正常工作

## 工具使用说明

### 主要工具
1. **new_page**: 创建新页面并导航到Swagger文档
2. **take_snapshot**: 获取页面快照，分析页面结构
3. **click**: 点击页面元素（展开模块、执行接口等）
4. **fill**: 填写接口参数
5. **list_network_requests**: 查看网络请求详情
6. **wait_for**: 等待页面元素加载

### 辅助工具
- **list_pages**: 查看当前打开的页面
- **select_page**: 切换到指定页面
- **take_screenshot**: 截图保存当前状态

```
2. 获取页面快照

```

```
AI:
```


## 注意事项

1. **网络环境**: 确保能够访问 http://172.30.0.28:31501/doc.html
2. **权限验证**: 如果接口需要认证，需要先登录获取token
3. **参数格式**: 填写参数时注意JSON格式和数据类型
4. **响应超时**: 某些接口可能响应较慢，需要适当延长等待时间
5. **错误处理**: 如果接口调用失败，需要分析错误原因并给出建议

## 常见问题处理

### 问题1: 页面加载失败
- 检查网络连接
- 确认服务地址是否正确
- 检查服务是否正常运行

### 问题2: 找不到指定模块
- 使用快照查看所有可用模块
- 建议用户选择相似模块或检查模块名称

### 问题3: 接口调用失败
- 查看网络请求详情
- 检查请求参数是否正确
- 检查是否需要认证token
- 查看响应错误信息

### 问题4: 参数填写困难
- 查看接口文档中的参数说明
- 使用示例数据作为参考
- 建议用户提供测试数据

## 扩展功能

### 批量测试
可以扩展为批量测试同一模块下的多个接口，生成测试报告。

### 自动化测试
可以结合测试用例，实现接口的自动化回归测试。

### 数据对比
对比不同环境下的接口响应数据，验证接口一致性。

## 技术要点

1. **页面元素定位**: 通过uid精确定位页面元素
2. **异步等待**: 使用wait_for确保页面加载完成
3. **网络监控**: 通过list_network_requests捕获接口调用
4. **快照分析**: 利用take_snapshot理解页面结构
5. **交互引导**: 逐步引导用户完成接口测试流程
