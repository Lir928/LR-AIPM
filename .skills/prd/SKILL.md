# PRD 需求工程技能

面向产品经理与研发协作场景，提供从创意到可执行规格的全流程支持：结构化 PRD 输出、需求拆分与优先级、验收标准、里程碑与风险、以及与原型协同的组件规范。

## 适用时机
- 用户提出新想法，需要快速生成初版 PRD
- 现有需求需拆分为可执行条目并定义验收标准
- 评审/补充 PRD 的遗漏与不一致
- 将 PRD 条目映射为基础交互原型（Axhub）所需的组件与事件

## 输入期望
- 目标/痛点（业务背景与成功指标）
- 目标用户与关键使用场景
- 已知约束（技术/合规/时间）
- 初步功能清单或关键页面

## 输出内容
- 标准化 PRD（可直接分享评审）
- 需求条目（MoSCoW 或 RICE）
- 场景与用户故事（Given-When-Then）
- 验收标准与边界条件
- 里程碑与风险清单
- Axhub 原型协作说明（组件与交互规范）

## 工作流程
1) 对齐目标：明确业务目标、成功指标、非目标范围  
2) 场景建模：关键用户、任务链路、边缘情形  
3) 需求拆分：产出条目与优先级，标注依赖与风险  
4) 验收定义：每条需求给出可验证的验收标准  
5) 原型协同：抽取页面/组件/事件，形成 Axhub 指南  
6) 路线安排：里程碑、版本范围与发布日期标尺

## Axhub 原型协同说明
- 页面结构：按功能域划分页面/区块
- 组件规范：按钮圆角 8px，主色 #10b981，仅内联样式；图片使用 https://picsum.photos/
- 交互事件：点击/悬停/输入反馈的触发条件与结果
- 状态控件：加载、空态、错误、权限受限
- 自适应：最小宽度策略与折行规则

## 使用指引
- 当用户说“把这个想法写成 PRD/细化需求/给出验收标准/准备原型规范”时，基于上述模板生成结构化文档
- 缺失信息时主动补齐假设，并用“假设”段落显式标注
- 输出应包含“下一步原型组件清单与事件映射”，便于快速进入 Axhub 搭建

## 示例调用提示

- “为这个核心场景拆分需求并标注优先级、边界条件：…”
- “把此 PRD 转成原型组件与页面清单，含交互事件与状态：…”
