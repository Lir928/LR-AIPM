# Axhub逻辑增强流水线

## 触发词

`执行原型增强`

## 流水线概述

在 04 流水线生成了基础的 Axhub UI 原型后，本流水线将调用高级研发专家技能，直接对 Axhub 原型代码进行深度重构，注入真实的 API 请求、复杂状态管理和真实的业务 CRUD 逻辑，使原型直接进化为具备真实生产能力的可用模块。

## AI 协同顺序与执行逻辑

- 【步骤 1：读取基础原型与 PRD】：主动定位并读取 `axhub-make/src/prototypes/[功能名称（英文简写）]/` 目录下的基础原型代码（`index.tsx` 等），同时读取对应的 PRD 文档获取真实的业务规则。
- 【步骤 2：专家技能介入】：强制调用 `.skills/` 里的高级研发专家（如 `@crud-implementation`、`@interaction-logic-architect` 等），根据 PRD 的状态机和数据字典，设计真实的前端交互逻辑。
- 【步骤 3：直接重写 Axhub 代码 (Hard Rule)】：将所有复杂的业务逻辑、状态管理（Hooks）、网络请求等，【直接写入并覆盖】 `axhub-make/src/prototypes/[功能名称（英文简写）]/` 下的代码文件！绝对禁止在其他任何地方创建独立的 src 目录！
- 【步骤 4：交付预览】：在聊天框向用户反馈："✅ 研发技能已成功注入！您的 Axhub 原型现已具备真实业务逻辑，请刷新浏览器 http://localhost:51724/prototypes/[功能名称英文简写] 体验完全体！"
