```
我要联调用户管理模块的接口
```

1. 打开 http://172.30.0.28:31501/doc.html

## 示例4: 错误处理场景

### 用户输入
```
测试删除用户接口，用户ID为99999
```

### 执行流程
1. 定位到"删除用户"接口
2. 点击"Try it out"
3. 填写路径参数：id=99999
4. 执行请求
5. 分析错误响应

### 预期输出
```
AI: 好的，我来测试删除用户接口。

[点击删除用户接口]
[点击Try it out]
[填写路径参数: id=99999]
[执行请求]

接口调用失败！
请求方法: DELETE
请求路径: /api/user/delete/99999

响应状态码: 404
响应数据:
{
  "code": 404,
  "message": "用户不存在",
  "data": null
}

错误分析：
- 状态码404表示资源未找到
- 错误信息：用户不存在
- 可能原因：用户ID 99999 不存在

建议：
1. 先调用获取用户列表接口，确认有效的用户ID
2. 使用存在的用户ID进行删除操作
3. 检查用户是否已被删除

需要我帮你查询有效的用户ID吗？
```
