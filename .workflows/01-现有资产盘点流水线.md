# 现有资产逆向学习流水线

## 触发词

`执行老项目盘点`

## 流水线概述

本流水线用于在启动新需求开发前，让 AI 深入读取并理解已有的代码库、数据库 Schema 和接口文档，建立项目全局上下文，生成逆向文档，确保后续生成的代码与既有架构完美契合。

## AI 协同顺序与执行逻辑

### 步骤 1：历史文档知识提取

- **技能**：@legacy-doc-analyzer
- **任务**：读取 `projects/[项目名称]/legacy-assets/` 目录下的各种非结构化文档（Word、Markdown、txt 等），提炼业务规则、专有名词（Glossary）和隐藏逻辑。
- **输出**：在 `projects/[项目名称]/docs/` 生成 `00-历史文档分析报告.md`。

### 步骤 2：存量接口解析

- **技能**：@api-doc-parser
- **任务**：解析 `projects/[项目名称]/legacy-assets/` 目录下的 API 接口文档，提取接口定义、参数说明和返回值结构。
- **输出**：在 `projects/[项目名称]/docs/` 生成 `00-接口分析报告.md`。

### 步骤 3：拓扑与架构扫描

- **技能**：@codebase-mapper
- **任务**：全局扫描 `projects/[项目名称]/legacy-assets/` 目录下的老代码和旧文档，梳理模块依赖关系。
- **输出**：在 `projects/[项目名称]/docs/` 生成 `01-系统现存架构图.md`。

### 步骤 4：数据资产逆向解析

- **技能**：@schema-reverse-engineer
- **任务**：提取 `projects/[项目名称]/legacy-assets/` 中的代码类型定义、建表 SQL 或老 API 文档，逆向出实体关系图。
- **输出**：在 `projects/[项目名称]/docs/` 生成 `02-现存数据字典.md`。

### 步骤 5：业务规则还原与基线确认

- **技能**：@reverse-prd-generator
- **任务**：结合上述步骤，总结出系统目前已经实现了哪些功能，梳理出存量 PRD，并圈出高风险代码区域。
- **输出**：在 `projects/[项目名称]/docs/` 生成 `03-存量业务逻辑基线.md`。

## 后续动作

本流水线执行完毕后，生成的五个文档将作为 `02-需求转PRD流水线` 和 `04-产品交付转代码流水线` 的强制前置上下文（Context）。AI 在生成新代码时，必须严格遵守老项目中已定义的数据结构和 UI 规范。
