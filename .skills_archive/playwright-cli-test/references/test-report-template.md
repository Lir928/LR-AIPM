# 测试报告模板

## 报告格式

```markdown
# {需求名称} 测试报告

## 执行概览

| 指标 | 数值 |
|------|------|
| 总用例数 | X |
| 通过数 | X |
| 失败数 | X |
| 通过率 | X% |
| 执行开始时间 | YYYY-MM-DD HH:mm:ss |
| 执行结束时间 | YYYY-MM-DD HH:mm:ss |
| 总耗时 | X分X秒 |

## 失败用例详情

### TC-{ID}: {用例名称}

**步骤**:
1. 步骤1
2. 步骤2

**实际结果**:
{描述实际发生的情况}

**截图/录屏路径**:
`docs/test/{TRACKING_ID}/screenshots/{用例ID}-{时间}.png`

**失败原因**:
{错误信息}

## 通过用例列表

| 用例ID | 用例名称 | 耗时 |
|--------|----------|------|
| TC-XXX | 用例名称 | Xms |
```

## 执行日志示例

```
[2024-01-01 10:00:00] 测试开始
[2024-01-01 10:00:05] 执行用例 TC-LOGIN-001
[2024-01-01 10:00:10] 用例 TC-LOGIN-001 通过
[2024-01-01 10:00:11] 执行用例 TC-LOGIN-002
[2024-01-01 10:00:15] 用例 TC-LOGIN-002 失败，错误信息：元素未找到
[2024-01-01 10:00:16] 重试用例 TC-LOGIN-002
[2024-01-01 10:00:20] 用例 TC-LOGIN-002 通过
[2024-01-01 10:00:21] 测试结束
```
