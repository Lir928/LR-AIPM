# 02-需求转PRD流水线

## 流程概述

本流水线将原始需求转化为详细的产品需求文档（PRD），包含完整的功能描述、页面设计、字段定义和交互流程。

## 步骤设计

### 步骤 1：需求分析与拆解
- 分析原始需求文档
- 拆解需求为具体功能点
- 识别核心业务逻辑
- 确定需要的页面和功能模块

### 步骤 2：架构设计
- 设计系统架构图
- 确定技术栈和依赖
- 规划数据模型
- 设计API接口

### 步骤 3：逻辑闭环设计
- 调用 `@requirements-to-detailed-design` 生成详细设计
- **强制调用 `@enterprise-product-designer` 应用企业级 B 端产品设计规范**
- 为每个核心页面绘制 ASCII 布局图
- 输出包含字段名、数据类型、校验规则的严谨表格
- 确保所有交互流程完整闭环
- 验证设计是否符合企业级规范

### 步骤 4：PRD 生成
- 整合所有设计内容
- 生成标准化的 PRD 文档
- 包含完整的页面设计、字段描述和交互流程
- 确保 PRD 可直接用于 Axhub 原型生成

### 步骤 5：质量检查
- 检查 PRD 的完整性和准确性
- 验证字段描述是否符合数据库设计规范
- 确认校验规则是否可直接用于前端验证
- 确保 ASCII 布局图准确反映页面结构
- 检查流程描述是否完整覆盖所有场景

## 输出要求

1. 标准化的 PRD 文档
2. 包含 ASCII 布局图的页面设计
3. 详细的字段级描述表格
4. 完整的交互流程描述
5. 符合企业级 B 端产品设计规范的输出格式

## 技术要求

1. 严格遵循企业级 B 端产品设计规范
2. 确保 PRD 可直接用于 Axhub 原型生成
3. 字段描述必须可直接用于数据库设计
4. 校验规则必须可直接用于前端验证
5. 所有输出必须符合格式规范
