# 项目启动与预览流水线

## 触发词

`启动[项目名称]`（如：启动基层组织工作台、启动demo-task）

## 流水线概述

本流水线用于一键切换目标项目的上下文，并自动调起 Axhub Make 本地引擎与浏览器进行可视化预览。

## AI 协同顺序与执行逻辑

当接收到触发词后，AI 助手必须主动调用内置终端 (Terminal)，按严格顺序执行以下两条命令：

1. `npm run switch [解析出的项目名称]`
2. `npm run dev:axhub-make`

## 后续动作

在聊天框向用户反馈："✅ [项目名称] 环境已激活，Axhub Make 引擎已启动并在浏览器中打开预览。"

## 执行条件

- 目标项目目录已存在于 `projects/` 目录下
- 网络连接正常
- Axhub Make 本地引擎已安装
- 已打开 `axhub-make/Axure-axhub-make引擎_RP11.rp` 文件以激活 Axhub Make 引擎

## 预期成果

- 目标项目环境已激活
- Axhub Make 引擎已启动
- 浏览器已自动打开并显示项目预览

## 注意事项

- 确保目标项目名称正确，且已存在于 `projects/` 目录下
- 如遇启动失败，检查项目目录结构是否完整
- 如需停止 Axhub Make 引擎，可在终端中按 Ctrl+C 终止进程
