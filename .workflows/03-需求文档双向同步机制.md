# 需求文档双向同步机制

## 触发词
`执行Prompt同步`

## 流水线概述
本流水线用于在用户修改需求文档后，自动同步更新对应的 Axhub Make 原型提示词，确保需求文档与原型提示词的一致性。

## AI 协同顺序与执行逻辑

### 步骤 1：读取需求文档
- **任务**：读取当前在编辑器中激活的 `YYYYMMDD-[功能名称].md` 文件。
- **输出**：读取的需求文档内容。

### 步骤 2：重新生成 Axhub 原型 Prompt
- **技能**：@requirements-closure-designer
- **任务**：根据修改后的最新文档内容，重新生成 Axhub 专属 Prompt。
- **输出**：更新后的 Axhub 专属 Prompt 文本。

### 步骤 3：覆盖更新 Prompt 文件
- **任务**：根据需求文档的文件名，确定对应的 Prompt 文件路径（`projects/[项目名称]/prompts/YYYYMMDD-[功能名称]-Prompt.md`），并覆盖更新该文件。
- **输出**：更新后的 Prompt 文件路径。

### 步骤 4：提示用户
- **任务**：提示用户更新完成，可以继续输入 `执行产研流水线` 进行开发。
- **输出**：提示信息。

## 预期成果
- 需求文档与 Axhub 原型提示词保持同步
- 用户可以在修改需求文档后，通过简单的触发词更新对应的 Prompt
- 确保开发过程中需求与原型的一致性

## 执行条件
- 需求文档已存在且已被修改
- 对应的 Prompt 文件已存在
- 网络连接正常

## 注意事项
- 确保需求文档的文件名格式为 `YYYYMMDD-[功能名称].md`
- 确保 Prompt 文件的路径为 `projects/[项目名称]/prompts/YYYYMMDD-[功能名称]-Prompt.md`
- 如果无法找到对应的 Prompt 文件，将提示用户创建新的 Prompt 文件
- 同步过程中会覆盖原有 Prompt 文件，确保用户已保存需求文档的修改
