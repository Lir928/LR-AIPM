# 用户操作手册自动编写

根据功能页面URL自动生成用户操作说明文档的完整工作流程。

## 工作流程

### 第一步：页面确认

1. 要求用户提供：
   - 功能名称
   - 页面URL地址

2. 使用 Playwright CLI 访问目标URL，确认页面可正常打开
3. 页面正常后进入下一步

### 第二步：全局分析

访问目标功能页面，进行全局截图和分析：

1. 使用 Playwright CLI 对页面整体截图
2. 分析输出：
   - **页面总体布局**：顶部导航、侧边栏、主内容区等
   - **独立功能点清单**：主页面、Tab页、弹窗、表单、列表等

3. **重要**：等待用户确认功能点清单
   - 如用户有调整意见，进行相应修改
   - 用户确认无误后再继续

### 第三步：截图获取

根据确认的功能点清单，逐个获取截图：

**截图要求：**
- 不遗漏任何独立功能点
- 截图命名规范：`功能页面名称_功能点名称_组件类型_可实现的功能.png`
- 示例：`模板管理_质量管理_弹窗_任务创建.png`

**保存路径：**
- 保存到 `操作手册编写/images/` 目录
- 每个功能页面创建独立子目录，目录名为功能页面名称

**需要截图的功能点类型：**
- 主页面整体
- Tab切换页面
- 弹窗（新建/编辑/详情等）
- 列表操作区域
- 表单区域
- 其他独立功能组件

### 第四步：内容生成

针对截图生成用户操作说明文档：

**文档结构：**

```markdown
# 功能名称

## 页面简介

- **页面路径**：菜单路径
- **功能描述**：简要描述
- **主要功能点**：列出核心功能
- **业务功能**：实现的业务目标

![主页面截图](images/功能名称/主页面.png)

## 功能点说明

### 功能点1名称

![功能点截图](images/功能名称/xxx.png)

**功能说明**：描述

**操作步骤**：
1. 步骤一
2. 步骤二

**字段说明**（如有表单）：
- **字段名**（必填/选填）：说明

**注意事项**（如有）：
- 注意点

### 功能点2名称
...
```

**内容组织原则：**
- 按业务流程顺序组织：浏览查询 → 新建编辑 → 其他操作
- 同一业务功能放在一起说明
- 每个功能点前先放截图
- 不需要说明分页功能（通用功能）

**功能点描述要素：**
- 功能基本说明
- 操作步骤
- 功能特点
- 表单字段说明（如有）
- 注意事项（如有）

## 输出示例

```markdown
## 新增任务功能

![新增任务弹窗](images/质量任务/质量任务_新增任务_弹窗_任务配置表单.png)

**新增任务配置说明**:
- **任务名称**（必填）：输入任务的显示名称
- **任务描述**：输入任务的详细描述信息
- **数据源名称**（必填）：从下拉列表中选择数据源
- **所含模型**（必填）：从下拉列表中选择质量检查模型
- **执行周期**：选择任务执行方式
  - 立即执行：任务创建后立即执行
  - 周期执行：按设定的周期定时执行
```

## 注意事项

1. 截图命名要完整反映截图信息，便于文档维护
2. 功能点分析要全面，不遗漏独立功能
3. 文档内容要等待用户确认后再继续下一步
4. 使用 Playwright CLI 的有头模式进行截图，确保页面可见
