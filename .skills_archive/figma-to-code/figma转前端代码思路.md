# AI 辅助UI转前端代码指南

为提升团队开发效率与设计还原度，建议按照以下五步流程，结合主流 AI 编程辅助工具（如 OpenCode、Trae、Claude 等）开展前端开发工作。

------

## 第一步：生成 AI 编程指导文档

使用 AI 工具（如 OpenCode、Trae、Claude 等）对当前前端项目框架进行深度解析，自动生成一份标准化的 AI 编程指导文档。  

- **文档命名建议**：`AGENT.md` 或 `CLAUDE.md`  
- 内容应包括但不限于：
  - 使用的编程语言与框架（如 JS + Vue 3）
  - UI 组件库（如 Ant Design、Element Plus 等）
  - 核心依赖与版本约束
  - 项目目录结构说明
  - 样式规范（Tailwind / SCSS 等）
  - 构建与部署流程简述

> 此文档将作为后续 AI 工具理解项目上下文的基础依据。

------

## 第二步：导入 Figma 设计资源

1. 在 Trae 中启用 **Solo 模式**。
2. 启用 **Figma 插件**。
3. 在 Figma 设计稿中选中目标页面或组件区域。
4. 点击插件中的 **“添加到对话”** 按钮。执行后，Trae 将自动下载所有相关切图资源及示例代码，并保存至项目根目录（通常为.figma的路径）。
5. 为节约ai编程的时间，需要提前将Figma下载的切图资源放在合理的目录下（通常为 src\assets\images\XXXX 下）。

------

## 第三步：编写定制化提示词

基于第一步生成的框架文档和第二步获取的设计资源，编写精准、结构化的提示词。  

- 可参考文末提供的通用模板；
- **务必根据本项目实际技术栈、组件规范和设计风格进行调整**；

------

## 第四步：创建 AI 工作计划（Plan 模式）

将第三步整理好的提示词输入至 AI 工具（如 OpenCode、Trae、Claude 等），并选择 **Plan 或 solo 模式**，让 AI 自动生成分步骤的开发任务计划。  

- 开发者可在此基础上进行审核、调整或直接执行。

------



## **技巧1：校验与视觉一致性调整**

**目标**：确保 AI 生成的前端代码与原始 Figma 设计稿在视觉表现和交互行为上高度一致。

### **操作步骤：**

1. **人工比对**
   将 AI 输出的界面与 Figma 设计稿逐项对比

2. **使用 Trae 浏览器模式精准修正**
   若发现偏差，推荐启用 **Trae 的浏览器插件**进行可视化调试：
   - 在浏览器中直接选中目标 DOM 元素；
   
   - 向 AI 发出具体、可量化的修改指令，例如：
   
     - “将主按钮的圆角从 4px 改为 8px”
   - “卡片内上下 padding 调整为 16px，左右为 24px”
     - “标题文字颜色改为 #1F2937，字体大小 18px”
   
   - AI 将自动定位对应代码（HTML/CSS/SCSS/Vue 组件），并完成精准修正。

------

## **技巧2：SKILLS 创建与智能复用**

**目标**：将重复性经验沉淀为可复用的 SKILLS，提升后续开发效率。

1. **创建 SKILLS：总结失败经验**

​	当某个问题多次修改才实现解决时，可以在解决后主动要求 AI 总结原因并创建 SKILLS：

提示词示例：

```
请总结刚才你修改 Element Plus 样式失败的原因，并将解决方案封装成一个名为 "override-element-plus-style" 的 SKILL。下次我要求修改 Element Plus 内置组件样式时，请自动调用该 SKILL。
```

2. **创建 SKILLS：组件使用指南**

​	对已开发完成的业务组件，可让 AI 自动生成标准化使用说明 SKILL，下次涉及调用组件实现的地方快捷生成一致性代码：

提示词示例：

```
分析 src/components/CommonDialog.vue 组件的 props、事件和插槽结构，生成一份名为 "use-common-dialog" 的 SKILL，包含调用示例和注意事项。
```

3. **调用 SKILLS：在提示词中显式引用**

​	在新任务中通过提及 SKILL 名称触发自动复用：

> 示例提示词：
> “该设计稿是弹窗中间内容的组成部分，弹窗标题为 **流域健康报告**。请复用 `CommonDialog` 组件实现弹窗。点击 `<div class="bg-list">` 时弹出该弹窗。

------

## **技巧3：图表高保真还原（基于 ECharts）**

**目标**：根据设计稿或截图，1:1 还原 ECharts 图表的视觉样式与交互功能。

**推荐提示词模板：**

> “请将当前占位元素替换为 ECharts 折线图。要求：
>
> - 完全复刻附件截图中的配色、坐标轴样式、图例位置、网格线、tooltip 样式；
> - 支持 hover 高亮、数据缩放、图例切换等标准交互；
> - 使用响应式配置，适配容器宽度；
> - 提供模拟数据结构示例。”

------

## **技巧4：复杂 Bug 调试策略**

**目标**：系统化解决难以定位或涉及多模块的复杂问题。

### **推荐流程：**

1. **启动计划模式**
   使用 `/plan` 指令让 AI 先输出调试方案，而非直接修改代码：

   > “/plan 我在使用 CommonDialog 时，点击遮罩层无法关闭弹窗。请先分析可能原因，并列出排查步骤。遇到不清楚的地方务必停止回答并等待我的确认”

2. **交互式确认**
   在计划阶段，AI 会主动提问澄清模糊信息：

   > “请问该 Dialog 是否需要设置了`close-on-click-modal` 属性？”

3. **分步执行**
   确认计划无误后，再逐项实施修复，避免盲目修改导致副作用。

> ⚠️ 适用场景：状态管理异常、异步加载错误、样式冲突、第三方库集成问题等。

------

## 技巧5：从 GitHub 下载并使用社区 SKILLS

**目标**：复用高质量开源 SKILLS，提升开发效率。

### 操作示例：

1. **下载 SKILL**
   从团队仓库获取常用技能包
2. **启用与调用**  

   - skills支持自动加载，AI 将识别以下能力：

     - `superpowers`：自动规范编程流程；
     - `ui-ux-pro-max`：优化UI设计。

> 2. 代码实现规范
>     figma代码仅供你参考，不得复用：Figma 中附带的示例代码仅用于理解布局结构，不可直接复制使用。
>     遵循项目规范：所有代码必须严格遵守根目录下 AGENTS.md 文件中定义的编码规范。
>     组件化拆分：必须将页面按功能和视觉模块拆分成多个独立、职责单一的组件文件，避免将所有逻辑和 UI 堆砌在单一文件中。确保代码结构清晰、命名语义化、逻辑解耦，保持高度的整洁性、可读性和可维护性。
